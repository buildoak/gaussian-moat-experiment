## **3. Theoretical considerations**

### **3.1 Percolation analogy**

In continuum percolation with disks radius r, critical density η_c ≈ 1.128 (for Poisson process in 2D).  
Gaussian primes density at norm N is ~ 1/(2 log N) in first octant, but in full plane it’s ~ 1/(log N) per area? Actually: #Gaussian primes with norm² ≤ X ~ X/log X (up to constant).  
So areal density at radius R is ~ (1/(πR²)) × (πR²/log R²) = 1/(2 log R) (points per unit area? Wait, that’s not right — careful).

Number in disk radius R ≈ R²/log R. Area = πR², so density = (R²/log R)/(πR²) = 1/(π log R). Yes, density → 0 as R→∞, but slowly.

Percation threshold in 2D Poisson Boolean model: λ_c ≈ 1.128. λ = mean number of points per area of πr². Here λ = (density) × πr² = (1/(π log R)) × πr² = r² / log R.

For percolation, need λ > λ_c for infinite cluster. Here r=k fixed, R increasing → λ decreases as 1/log R, so eventually λ < λ_c → no percolation. This is why moats exist for any fixed k.

But here we ask: given k, how far can we go before λ drops below λ_c locally? That’s not exactly right — we only need connectivity to origin, not infinite cluster globally.

Better: think of branching process. Each prime has expected number of neighbors within distance k ≈ density × πk² = (πk²)/(π log R) = k²/log R. For branching to survive, need k²/log R > 1 (mean children > 1). So critical R satisfies log R ≈ k².

For k²=40 → log R ≈ 40 → R ≈ e⁴⁰ ≈ 2.35×10¹⁷ — huge! But we observe R_moat(36) < 10⁸. So mean children > 1 condition is too weak — clustering matters, and finite-size effects kill the component earlier.

---

### **3.2 Origin-lineage extinction mechanism**

The component dies not because density is too low everywhere, but because **forward growth** (primes with larger norm) have fewer connections back to the cluster. At each step, new primes are found, but they might not connect to any prime in the current cluster — they form separate islands. The existing cluster stops growing when all frontier primes have no neighbors ahead within step k.

This depends on **local gaps** in prime distribution at scale ~k. Gaussian primes are distributed like primes p=4m+1 (points on circle) and p=4m+3 (on axes). The p=4m+1 ones are more relevant off-axis. Their average spacing at norm N is ~ log N. If k < log R, then gaps larger than k become likely, causing disconnection.

For R ~ 10⁸, log R ~ 18.4, k=6, so k < log R indeed. So isolated islands form.
