## Task: Implement spec-aligned dead-end pruning in tile-cpp C++

### Context

The C++ tile processor at `tile-cpp/` has all 5 phases working (sieve, compact, union-find, face extract, encode) and is aligned with the 257x257 shared-boundary convention. However, it does NOT implement dead-end pruning. The Python validator does, which causes group count divergence:

| Tile | C++ groups | Python groups |
|------|-----------|---------------|
| 45 deg | 27 | 11 |
| 30 deg | 27 | 16 |
| 15 deg | 14 | 6 |

Your job: add spec-faithful dead-end pruning to the C++ pipeline.

### Spec reference

### Dead-end pruning spec

A group is a "dead end" if it touches exactly ONE face with exactly ONE port on that face. Such groups cannot contribute to spanning paths (they enter from one face and have no exit to another). They are removed before encoding.

After pruning, surviving groups are renumbered contiguously starting at 1 (group 0 is reserved as empty sentinel).

### Implementation plan

1. **Read the current pipeline** in `tile-cpp/src/process_tile.cpp` — understand the flow between face extraction and encoding.

2. **Add a pruning phase** between face extraction and encoding. Create `tile-cpp/src/prune.cpp` and `tile-cpp/include/prune.h` (or add to an existing file if more natural).

3. **Pruning algorithm:**
   - For each group, count: how many faces does it appear on, and how many ports does it have on each face?
   - A group is dead-end if: it appears on exactly 1 face AND has exactly 1 port on that face
   - Remove all dead-end groups: delete their ports from the face lists
   - Renumber surviving groups contiguously starting at 1
   - Update group_count to reflect only surviving groups

4. **Wire into process_tile.cpp** — call pruning after face extraction, before encoding.

5. **Update CMakeLists.txt** if adding new source files.

6. **Update TileResult** — `group_count` should reflect post-pruning count. Add `ports_before_pruning` and `ports_after_pruning` fields if not already present.
