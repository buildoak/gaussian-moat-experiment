## Task: Migrate all specs and AGENTS.md to 257x257 tile proper (shared boundary convention)

### Background

A soundness audit proved that half-open 256x256 tiles with disjoint face-prime sets break positional slot matching in composition. The fix: tiles own 257x257 lattice points (0..256 per axis), adjacent tiles SHARE boundary lattice points. This is Convention B.

See the soundness audit at `docs/supportive/2026-04-09-convention-a-soundness-audit.md` for the counterexample.

### Key changes

S = 256 lattice UNITS = 257 lattice POINTS per axis (fencepost: 256 segments, 257 endpoints).

- Tile proper: 257 x 257 = 66,049 lattice points per tile
- `in_tile` predicate: `0 <= tile_row <= 256` (or equivalently `0 <= tile_row < 257` or `< S+1`)
- SIDE_EXP = 257 + 2*7 = 271 (the +1 in the old formula was the tile proper fencepost all along)
- Adjacent tiles share boundary: tile A at a_lo owns [a_lo, a_lo+256], tile B at a_lo+256 owns [a_lo+256, a_lo+512]. Point a_lo+256 is in BOTH tiles.
- Face O of tile A and face I of tile B extract from the SAME lattice points on the shared boundary column/row
- Composition uses identity matching for I/O faces (shared primes appear in both tiles)
- Stride between tile origins remains S = 256

A tile of side S = 256 lattice units contains **257 x 257 lattice points** (fencepost: 256 segments = 257 endpoints per axis). Tile proper covers closed interval [0, S] per axis.

Adjacent tiles SHARE boundary lattice points. Tile A at origin a_lo owns [a_lo, a_lo + S]. Tile B at origin a_lo + S owns [a_lo + S, a_lo + 2S]. The column/row at a_lo + S belongs to BOTH tiles.
