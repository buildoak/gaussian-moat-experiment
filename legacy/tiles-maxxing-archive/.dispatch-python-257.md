## Task: Align Python tile-validator with 257x257 shared-boundary convention

### Context

The specs have just been updated to the 257x257 shared-boundary convention. The Python tile-validator at `tile-validator/` still uses the old 256x256 half-open convention. You must update it.

Read `AGENTS.md` first -- it has the hard rule. Then read the updated specs:
- `docs/tile_spec.md`
- `docs/tile_operations.md`

### What changed in the specs

1. Tile proper is now [0, S] inclusive per axis = 257 x 257 = 66,049 lattice points (was [0, S) = 256x256)
2. `in_tile` check: `0 <= tile_row <= S` (was `< S`)
3. SIDE_EXP = (S+1) + 2*collar = 257 + 14 = 271 (formula changed, value unchanged)
4. Face extraction ranges:
   - Face I: tile_row in [0, collar-1] = [0, 6] (7 rows, unchanged)
   - Face O: tile_row in [S - collar + 1, S] = [250, 256] (7 rows; was [249, 255])
   - Face L: tile_col in [0, collar-1] = [0, 6] (7 rows, unchanged)
   - Face R: tile_col in [S - collar + 1, S] = [250, 256] (7 rows; was [249, 255])
5. Adjacent tiles share the boundary row/column at tile_row = S (= 256)
6. I/O composition uses shared-prime identity matching (shared boundary primes appear in both tiles)

### What to change in Python

1. **tile.py**: Update `in_tile` predicate from `< TILE_SIDE` to `<= TILE_SIDE` (or `< TILE_SIDE + 1`). The sieve window (271x271) does NOT change size. The SIDE_EXP value stays 271.

2. **ports.py**: Update face extraction boundaries:
   - Face I: `tile_row < COLLAR` (unchanged, rows 0..6)
   - Face O: `tile_row > TILE_SIDE - COLLAR` (changed from `>= TILE_SIDE - COLLAR`; now rows 250..256 instead of 249..255). OR more precisely: `tile_row >= TILE_SIDE - COLLAR + 1` which is `>= 250`.
   - Face L: `tile_col < COLLAR` (unchanged)
   - Face R: `tile_col > TILE_SIDE - COLLAR` (same change as O)
   
   IMPORTANT: Be careful with the face boundary formulas. Read the updated tile_operations.md for the exact definitions. The key insight: with shared boundary, the O face's innermost row is now S - collar + 1 = 250 (was S - collar = 249). This keeps the face exactly collar=7 rows deep.

3. **compose.py**: Update I/O composition from positional slot matching to shared-prime identity matching. The shared boundary primes appear in both tiles' face-prime lists. Match by finding common (a, b) positions. Note: you may need to store prime positions in the TileOp or keep them as metadata alongside the 128-byte encoding.

4. **Constants**: If there are any constants like `TILE_POINTS = 256` or `BITMAP_SIZE = 256*256`, update to 257 / 257*257 as appropriate. But SIDE_EXP stays 271.
