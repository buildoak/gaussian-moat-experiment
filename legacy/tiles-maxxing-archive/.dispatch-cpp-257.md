## Task: Align tile-cpp C++ code with 257x257 shared-boundary convention and fix all bugs

### Context

The specs have been updated to the 257x257 shared-boundary convention. The C++ tile-cpp at `tile-cpp/` still uses the old 256x256 half-open convention. You must update it to match the specs.

Read these files FIRST:
- `AGENTS.md` -- hard rules including the 257x257 boundary convention
- `docs/tile_spec.md` -- updated spec
- `docs/tile_operations.md` -- updated spec
- `docs/supportive/2026-04-09-convention-a-soundness-audit.md` -- why half-open is unsound

### What changed

1. Tile proper is now [0, S] inclusive per axis = 257 x 257 lattice points (was [0, S) = 256x256)
2. `in_tile` check: `tile_row <= TILE_SIDE` and `tile_col <= TILE_SIDE` (was `< TILE_SIDE`)
3. SIDE_EXP = (S+1) + 2*collar = 271 (value unchanged, formula meaning changed)
4. Face extraction ranges:
   - Face I: tile_row in [0, COLLAR-1] = [0, 6] (unchanged)
   - Face O: tile_row in [TILE_SIDE - COLLAR + 1, TILE_SIDE] = [250, 256] (was [249, 255])
   - Face L: tile_col in [0, COLLAR-1] = [0, 6] (unchanged)
   - Face R: tile_col in [TILE_SIDE - COLLAR + 1, TILE_SIDE] = [250, 256] (was [249, 255])
5. Adjacent tiles share boundary lattice points
6. MAX_PRIMES may need adjustment (66049 tile-proper points vs old 65536)

### Specific bugs to fix

1. **in_tile check** in face_extract.cpp: `tile_row >= TILE_SIDE` rejects row 256. Change to `tile_row > TILE_SIDE` (reject row 257+, keep 0..256).

2. **Face O boundary**: currently `tile_row >= TILE_SIDE - COLLAR` which gives rows 249-255. Change to `tile_row >= TILE_SIDE - COLLAR + 1` which gives rows 250-256.

3. **Face R boundary**: same change as Face O but for columns.

4. **FACE_I/FACE_O comments in constants.h**: FACE_I is labeled "inner (top)" and FACE_O as "outer (bottom)". Fix to: FACE_I = "inner (bottom)", FACE_O = "outer (top)".

5. **MAX_PRIMES**: tile proper is now 257x257 = 66049 points. If MAX_PRIMES was sized for 256x256, check if it needs bumping. Current value is 16384 which is well above expected prime counts (~2000 at R=850M), so likely fine.

6. **Test coordinates**: test_e2e.cpp has tiles at (0,0), (100,100), (10000,10000). These violate AGENTS.md (R >= 800M). Replace with operating-point coordinates:
   - Tile A (45 deg): (601040640, 601040640)
   - Tile B (30 deg): (736121088, 424999936)
   - Tile C (15 deg): (820888320, 220000000)
