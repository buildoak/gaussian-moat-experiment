## Task: Prove or disprove mathematical soundness of Convention A (half-open 256x256 tiles) for Gaussian prime moat proofs

You are a mathematician auditing the soundness of a tiling system for proving Gaussian prime moats.

### Setup

The Gaussian moat problem: can you walk from the origin to infinity on Gaussian primes with bounded step size k (where k^2=40, so step distance <= sqrt(40) ~ 6.32)?

We tile the Gaussian integer lattice with square tiles to decompose this global connectivity question into local per-tile computations + a compositor that merges results.

### Convention A (the one used in the code and specs)

- Tile side S = 256
- Tile at origin a_lo owns lattice points [a_lo, a_lo + 256) per axis (half-open) = 256 points per axis
- Adjacent tile at a_lo + 256 owns [a_lo + 256, a_lo + 512) = 256 points
- **The two tiles do NOT share any lattice points**
- Tile A's rightmost owned column: a_lo + 255
- Tile B's leftmost owned column: a_lo + 256
- Gap of 1 lattice unit between outermost owned points of adjacent tiles
- Each tile has a collar/halo of 7 lattice points extending past the tile boundary
- Tile A's sieve domain: [a_lo - 7, a_lo + 263] (271 points, closed interval)
- Tile B's sieve domain: [a_lo + 249, a_lo + 519] (271 points)
- Sieve domain overlap: [a_lo + 249, a_lo + 263] = 15 lattice points
- Union-find runs on ALL sieved primes (including halo primes) within the 271x271 window
- Face primes extracted from tile proper only (within collar distance of boundary)
- Composition matches ports on abutting faces using positional slot matching

### The soundness question

For a moat proof, we need: if the tile system says "no spanning path from inner ring to outer ring exists," then no such path exists in the actual Gaussian prime graph.

In other words: the tile system must not MISS any real connection. It may report false connections (conservative), but it must never claim disconnection when a path actually exists.

### Specific concerns to address

1. **The 1-unit gap.** Tile A's face O has primes at tile rows 249-255 (global a_lo+249 to a_lo+255). Tile B's face I has primes at tile rows 0-6 (global a_lo+256 to a_lo+262). Consider a prime P1 at global position (a_lo+255, y1) in tile A and a prime P2 at global position (a_lo+256, y2) in tile B. They are 1 unit apart in the x-direction. If |y1 - y2| <= 6, they are within step distance sqrt(40). This connection P1-P2 exists in the real Gaussian prime graph. **Does the tile system detect it?**

   - P1 is in tile A's tile proper (row 255). It is a face prime on face O.
   - P2 is in tile B's tile proper (row 0). It is a face prime on face I.
   - P2 is also in tile A's HALO (tile A's sieve domain extends to a_lo+263, so P2 at a_lo+256 is visible).
   - P1 is also in tile B's HALO (tile B's sieve domain starts at a_lo+249, so P1 at a_lo+255 is visible).
   - Both tiles' UF processes both primes. Do both tiles correctly merge P1 and P2 into the same component?
   - If yes: P1's port on A's face O and P2's port on B's face I should be in connected components. Does the compositor correctly union them?

2. **Transitive connections through the gap.** Consider three primes: P1 at (a_lo+254, y1) in tile A, P_bridge at (a_lo+258, y2) in tile B's territory (but in A's halo), and P2 at (a_lo+262, y3) in tile B. If P1-P_bridge and P_bridge-P2 are each within step distance, then P1 and P2 are transitively connected. P1 is a face prime in A. P2 is a face prime in B. P_bridge is a halo prime for A and a tile-proper prime for B. **Does the compositor correctly detect P1-P2 connectivity?**

3. **Port matching soundness.** Convention A uses positional slot matching: port[i] on A's face O matches port[i] on B's face I. This assumes that the port ordering is consistent between the two tiles. But the two tiles compute their ports INDEPENDENTLY. Can the port ordering ever disagree, causing a mismatch?

4. **The deep question.** Is there ANY configuration of Gaussian primes near a tile boundary where Convention A's tile system claims "no connection across this boundary" but a step-connected path actually exists? This would be a soundness failure.
