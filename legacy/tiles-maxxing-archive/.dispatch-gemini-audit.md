## Task: Spec-compliance audit of the rewritten tile-validator Python module

You are a third-opinion auditor. A Codex worker just rewrote the Python tile validator at `tile-validator/` to align with two authoritative specs. Your job is to verify that the rewrite is faithful to the specs. You are NOT the implementer -- you are the auditor.

### Specs (read cover to cover)
- `docs/tile_spec.md`
- `docs/tile_operations.md`
- `AGENTS.md` (rules and authority chain)

### Python files to audit (read all)
- `tile-validator/tile.py`
- `tile-validator/ports.py`
- `tile-validator/uf.py`
- `tile-validator/compose.py`
- `tile-validator/validate.py`
- `tile-validator/primes.py`

### What to check

For EACH of the 7 alignment items, verify:

1. **Coordinate conventions** -- Does Python use bottom-left origin? Are faces named correctly (I=bottom, O=top, L=left, R=right)?

2. **Compact phase** -- Does Python build bitmap, prefix-popcount, and dense prime_pos array per tile_operations.md Phase 2?

3. **Union-find** -- Is it smaller-root-wins with full path flattening? No rank array? Per tile_operations.md S6.2/S6.4?

4. **Face extraction** -- Are halo primes excluded? Are only in-tile primes within collar of boundary used? Per tile_operations.md S7.1?

5. **Port clustering** -- Uses distance^2 <= k (2D Euclidean)? NOT 1D along-face separation? Per tile_operations.md S7.2?

6. **Dead-end pruning** -- Groups touching exactly one face with one port removed? 1-based group numbering? Overflow = all 128 bytes 0xFF? Per tile_spec.md S4.5, S4.6?

7. **128-byte TileOp encoding** -- Correct byte layout per tile_spec.md S4.1? I/O/L/R groups in bytes 0-63? L/R h1 in 64-95? Reserved zero in 96-127?

8. **Composition** -- Works on encoded bytes, not Python objects? I/O by slot index? L/R by h1+delta_h? Fingerprint demoted to diagnostic?

### Also check for bugs

- Off-by-one in face boundary calculations (collar inclusion/exclusion)
- Integer overflow risks in norm computation at R=850M
- Incorrect distance metric (taxicab vs Euclidean)
- Group 0 used as a real label instead of empty sentinel
- Port ordering within faces (spec may require specific order)
- Missing edge cases: empty tiles (no primes), single-prime tiles, all-overflow tiles
